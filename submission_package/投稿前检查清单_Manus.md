# 论文投稿前检查清单 (Pre-submission Checklist)

> **目标期刊**：《计算机工程与应用》
> **当前状态**：冲刺收尾阶段（完成度 ~95%）

在将论文正式提交至期刊投稿系统前，请务必逐项核对以下清单。

## 🔴 必检项（Fatal Errors - 影响初审通过）

- [ ] **标题与作者信息**：主稿顶部是否已补充正式标题、作者姓名、单位全称、邮编？
- [ ] **中英文摘要**：是否已补充英文标题、作者拼音、英文单位和 Abstract？
- [ ] **中图分类号**：是否已在关键词下方添加中图分类号（如 `TP242`）和文献标志码（`A`）？
- [ ] **章节去重**：第1章1.2节与第2章的文献综述部分是否已合并，消除了内容重叠？
- [ ] **引用顺序**：正文中的参考文献引用是否严格按照 [1], [2], [3]... 的顺序首次出现？（当前 [10][11] 先于 [9] 出现，必须修复）
- [ ] **数据一致性**：第4.7节声明的"均值 ± 标准差"是否已在第5章的表格中补齐？（或修改4.7节的表述）
- [ ] **图表编号**：全文图表是否已改为连续编号（图1、图2、表1、表2），去除了章节号（如"表 5-1"）？
- [ ] **双语图题**：所有图表的标题是否已补充英文对照？

## 🟡 风险项（Risk Factors - 易受审稿人攻击）

- [ ] **创新性表述**：摘要和引言中是否明确界定了本文与现有"加权A*"和"路径平滑"研究的差异？（建议强调：**针对8邻域真实地图的自适应权重映射函数**与**带碰撞检测的两阶段平滑**的系统性整合）。
- [ ] **对比基线**：本文仅对比了标准 A* 和消融变体，未对比 JPS 或 DWA。是否已在正文中合理防御？（建议在讨论部分说明：本文聚焦于 A* 算法本身的底层改进，改进后的算法可作为全局规划器与 DWA 等局部规划器无缝结合）。
- [ ] **运行时间绝对值**：运行时间在 0.04ms 级别，极短。是否已在实验设置中说明这是单次路径搜索的纯算法耗时，不包含地图加载和可视化开销？
- [ ] **"显著提升"话术**：全文是否已排查并删除了无数据支撑的"显著提升"、"极大改善"等空泛词汇？（当前版本已较好控制，但提交前需最后通读）。

## 🟢 可选项（Bonus Points - 提升录用概率）

- [ ] **开源代码声明**：是否在结论或引言中明确给出了 GitHub 仓库链接，声明代码和数据完全开源？（这在计算机类期刊中是极大的加分项）。
- [ ] **基金项目**：是否在首页脚注添加了相关的国家级或省部级基金资助信息？（核心期刊通常偏好有基金支持的论文）。
- [ ] **作者简介**：是否在文末补充了第一作者和通信作者的简介（出生年、性别、学位、研究方向、邮箱）？
- [ ] **高清图表**：提交的 Word 文档中的图表是否清晰？（建议将 `submission_package/figures/` 中的原图作为附件单独上传）。

---
**使用方法**：请在本地复制此清单，每完成一项打一个勾 `[x]`。当所有红色必检项和黄色风险项全部打勾后，即可放心投稿。
