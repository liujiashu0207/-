## 2. 实验数据深度剖析

我拉取了你最新跑完的 15 张真实地图（DAO/Street/WC3）的严格基准数据，并对 `improved_astar` 与传统 `astar` 进行了全面对比。

* **转弯次数（平滑度）**：在所有 15 张地图上实现断崖式下降，平均减少 **98.4%**（例如从 112 次转弯降至 1~2 次）。
* **结论**：你的平滑后处理（冗余点删除 + 拐角插值）效果极佳，**这足以支撑起一篇论文的“路径质量优化”部分**。

### 2.2 致命的劣势（会被拒稿的硬伤）
* **运行时间**：在所有 15 张地图上，改进算法的运行时间都比传统 A* 慢，平均慢了 **304.4%**。
* **扩展节点数**：平均增加了 **58.3%**，说明算法在搜索时做了大量无用功。

### 2.3 揪出罪魁祸首：消融实验分析
我分析了你的消融实验（Ablation）数据，找到了让算法变慢的“元凶”：
* **完整改进A*** 运行时间均值：`0.2044 ms`
* **去掉 JPS-like 后** 运行时间均值：`0.0810 ms`
* **去掉 自适应$\alpha$ 后** 运行时间均值：`0.0643 ms`
* **纯传统 A*** 运行时间均值：`0.0509 ms`

**真相大白**：你的 `JPS-like` 策略不仅没有加速，反而让算法慢了 152%！这是因为你只增加了搜索步长（导致邻居节点从 8 个暴增到 24 个），但没有实现 JPS 的核心——“强迫邻居剪枝”。优先队列被大量无效节点撑爆了。

### 第二步：修复自适应 $\alpha$（解决欠估计）
* **操作**：修改 `adaptive_alpha`，将其下界从 `0.8` 提升到 `1.0`。
  ```python
  def adaptive_alpha(obs_ratio: float) -> float:
      obs_ratio = min(max(obs_ratio, 0.01), 0.99)
      raw = abs(math.log(obs_ratio))
      return min(max(raw, 1.0), 1.8)  # 确保不会慢于传统A*
  ```
* **理由**：当 $\alpha < 1.0$ 时，A* 会退化为次优搜索，这是它目前比传统 A* 慢的另一个原因。
