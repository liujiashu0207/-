## 1. 研究问题（冻结）
在室内栅格静态环境中，改进 A*（Octile 启发 + 障碍率自适应权重 + JPS 融合 + 两阶段平滑）是否能在路径长度、转弯次数、运行时间上稳定优于传统方法。

## 2. 算法分组（冻结）

### 2.1 基线组
1. Dijkstra  
2. 传统 A*  
3. JPS  
4. 加权 A*（固定权重，无平滑）

### 2.2 改进组（Ours）
- 改进 A*（Octile + 自适应权重 + JPS 融合 + 两阶段平滑）

### 2.3 消融组（投稿建议必须做）
- Ablation-1：去掉自适应权重（保留固定权重）
- Ablation-2：去掉 JPS 融合
- Ablation-3：去掉路径平滑

## 3. 场景与地图（冻结）

### 3.1 地图规模
- 小图：20x20
- 中图：40x40
- 大图：80x80
- 超大图：100x100

### 3.2 障碍率分组
- 低障碍：20%~30%
- 中障碍：40%~50%
- 高障碍：60%~70%

### 3.3 每组实验轮次
- 每个“地图规模 x 障碍率”组合运行 30 次
- 固定随机种子列表，保证可复现

## 4. 评价指标与统计口径（冻结）

### 4.1 主指标（必做）
1. 路径长度（Path Length）
2. 转弯次数（Turn Count）
3. 运行时间（Runtime, ms）

### 4.2 辅指标（建议）
1. 扩展节点数（Expanded Nodes）
2. 成功率（Success Rate）

### 4.3 统计口径
- 每个指标报告：均值 +- 标准差
- 算法间进行显著性检验（t 检验，p < 0.05 作为显著）

## 5. 参数初值（先用，后调）
- 邻域：8 邻域
- 启发函数：Octile
- 固定权重基线：w = 1.2
- 自适应权重：alpha = f(obstacle_ratio)
- 路径平滑：先可视线删点，再圆弧/样条平滑

4. 参数记录：`config/params_v1.yaml`（或 markdown 表）
