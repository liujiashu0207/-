# 结论-证据对照表 v1

## 目的
把论文结论逐条绑定到可复查数据，避免“结论先行”。

## 数据来源
- 单种子实验：`results/exp_summary.csv`、`results/exp_significance.csv`
- 多种子复验：`results/exp_multiseed_significance.csv`

## 对照清单

| 编号 | 结论表述 | 证据定位 | 审核结果 |
|---|---|---|---|
| C1 | 改进A*在中大图低障碍（80/100, 0.3）对传统A*运行时间更低 | `exp_multiseed_significance.csv`，`baseline=astar, metric=runtime_ms, size in {80,100}, ratio=0.3, q<0.05` | 通过 |
| C2 | 改进A*在各场景相对传统A*转弯次数更少 | `exp_multiseed_significance.csv`，`baseline=astar, metric=turn_count`，8个场景均 `q<0.05` | 通过 |
| C3 | 改进A*路径长度更短（均值层面） | `exp_summary.csv` 中 `path_length_mean` 对比 `astar`；`exp_multiseed_significance.csv` 同指标 `q>=0.05` | 仅均值趋势，不可写“显著” |
| C4 | 改进A*一定比加权A*更快 | `exp_multiseed_significance.csv`，`baseline=weighted_astar, metric=runtime_ms` 若 `improvement_percent<0` | 不成立（部分场景更慢） |
| C5 | 改进A*在速度与路径质量间更均衡 | 时间与转弯在多场景显著，路径长度均值更优；结合 `summary+multiseed_significance` | 条件成立（需写“均衡”，不写“全指标显著”） |
| C6 | 实验可复现且样本量固定 | `results/exp_raw_records.csv` 行数1200；`exp_sampling_trace.csv` 显示采样追踪；`effective_repeats=30` | 通过 |

## 写作约束（强制）
1. 可以写“显著”的仅限：  
   - 运行时间（部分场景）  
   - 转弯次数（多数/全部场景）
2. 路径长度只能写“均值更优趋势”，不能写“统计显著更优”。
3. 与 `weighted_astar` 的速度对比必须写明“并非全场景更快”。
