## 对标维度（按审稿视角）

| 维度 | 对标论文做法 | 我们当前状态 | 结论 |
|---|---|---|---|
| 问题定义清晰度 | 明确三痛点（节点冗余、耗时、转角大） | 已在 `方法章节草稿_v1` 明确四模块目标 | 持平 |
| 方法链完整性 | Octile + 自适应权重 + 双向JPS + 二次平滑 | Octile + 自适应权重 + JPS-like + 二次平滑 | 略弱（JPS实现层级） |
| 对比基线 | A*、JPS | A*、weighted A*、JPS-like、Dijkstra | 更强 |
| 场景覆盖 | 多尺度 + 不同障碍率 + 实机 | 多尺度 + 不同障碍率（当前静态仿真） | 略弱（缺实机） |
| 指标体系 | 节点、路径、转弯、时间 | 节点、路径、转弯、时间 + 成功率 | 持平偏强 |
| 统计严谨性 | 主要报告均值趋势 | 置换检验 + FDR多重校正 + 多种子复验 | 更强 |
| 可复现性 | 文中描述平台，代码不完全公开 | 固定种子、满样本、采样追踪日志、脚本可复跑 | 更强 |
| 结论约束 | 主要趋势性描述 | 已建立“结论-证据对照表”约束写法 | 更强 |

## 自审结论（第1轮）
1. 我们在“统计与复现”维度已高于对标论文。  
2. 我们当前短板在“算法实现深度”与“实机验证”两点。  
3. 若要达到投稿更稳状态，优先补齐：
   - 把 `JPS-like` 升级为标准 JPS/双向JPS；
   - 增加最小实机或 ROS 回放验证一组。
