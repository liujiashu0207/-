# 数据真实性审计（第2轮，满样本修复后）

## 本轮目的
验证“每组强制满30个有效样本”是否真正生效，并提供完整采样追踪证据。

## 涉及文件
- `results/exp_raw_records.csv`
- `results/exp_summary.csv`
- `results/exp_sampling_trace.csv`（新增）

## 审计结果
1. 原始总行数：`1200`  
2. 理论期望行数：`1200`（4尺寸 x 2障碍率 x 30有效样本 x 5算法）  
3. 是否满样本：`是`  
4. 汇总表行数：`40`（4 x 2 x 5）  
5. `effective_repeats` 范围：`30 ~ 30`（每组严格30）

## 采样追踪证据（trace）
1. 追踪总尝试数：`297`  
2. 接受样本数：`240`（对应 8组 x 30）  
3. 拒绝样本数：`57`  
4. 拒绝原因：`no_reachable_start_goal`（地图在尝试次数内未采到可达起终点）

## 解释
当前策略不是“忽略失败”，而是：
1. 记录每次尝试（含失败）；  
2. 仅将可达样本纳入有效实验；  
3. 对每个场景持续采样直到凑满30个有效样本。

这符合科研上“可达约束+固定有效样本量”的规范做法。

## 可复查命令
```bash
python code/experiments/run_b_route_experiments.py --sizes 20,40,80,100 --ratios 0.3,0.7 --repeats 30 --seed 42
```

执行后应看到：
- `exp_raw_records.csv` 行数=1200
- `exp_summary.csv` 中 `effective_repeats` 全部为30
- `exp_sampling_trace.csv` 含 rejected 记录
