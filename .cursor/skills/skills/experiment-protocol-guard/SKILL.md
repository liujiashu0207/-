# Experiment Protocol Guard

## 目标
保证“同一论文只用一套口径”，避免口径漂移导致结论失效。

## 触发词
- 复现实验、协议、strict scen、repeats、seed、基线、消融
- run_fix15_v3、exp_fix15_v3、权威结果、主口径

## 守卫清单
- 只认主实验脚本与主结果前缀（例如 v3）。
- 起终点来源必须与协议一致（如 `.scen` 固定任务）。
- 重复次数、地图数、算法集合必须完整。
- 参数值必须与文档一致（如 weighted A* 的 weight）。
- 输出文件路径与命名必须可追溯。

## 执行步骤
1. 读取协议文档与主实验脚本，提取口径字段。
2. 对比结果文件头字段与关键统计值是否匹配。
3. 检查文档/图例是否与代码参数一致。
4. 输出“通过项/问题项/修复建议”三段式结论。

## 输出模板
```markdown
## 协议验收结果
- 总结：通过 / 部分通过 / 不通过

### 通过项
- ...

### 问题项
- [严重级别] 描述（文件）

### 修复建议
1. ...
2. ...
```

## 质量门禁
- 发现参数口径冲突时，默认判为必须修复。
- 未能证明可复现时，不给“可投稿”结论。
