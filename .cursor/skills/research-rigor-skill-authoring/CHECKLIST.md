# 科研输出检查清单

在交付前逐项检查：

- [ ] 所有关键结论都绑定了证据锚点。
- [ ] 没有把未知信息写成确定事实。
- [ ] 没有把相关性直接表述为因果关系。
- [ ] 给出了可复现信息（数据版本、参数、环境、随机种子）。
- [ ] 指标定义清晰且口径一致。
- [ ] 涉及“显著提升/差异”时提供了统计检验信息。
- [ ] 明确列出局限、偏差与外推风险。
- [ ] 对证据不足部分使用“待验证结论”。
- [ ] 术语前后一致（证据锚点、可复现信息、待验证结论）。
- [ ] 输出语言与用户要求一致（默认中文）。

# Skill 编写检查清单

- [ ] `SKILL.md` 包含合法 frontmatter（`name`、`description`）。
- [ ] `name` 为小写+连字符，长度 <= 64。
- [ ] `description` 明确 WHAT + WHEN，并包含触发词。
- [ ] `SKILL.md` 不超过 500 行。
- [ ] 引用文件保持一层深度（例如 `TEMPLATE.md`、`CHECKLIST.md`）。
- [ ] 无时间敏感且易过期的硬编码规则。
- [ ] 术语统一，无歧义命名。
