# 第3章 方法（草稿 v1）

## 3.1 问题定义与符号
本文在二维栅格地图中研究移动机器人全局路径规划问题。地图中每个栅格仅有“可通行”和“障碍”两种状态，机器人在 8 邻域内移动。设起点为 `s`，终点为 `g`，路径记为 `P={p0,...,pk}`。目标是在无碰撞前提下，兼顾路径长度、转弯次数与规划时间。

传统 A* 的评价函数为：

\[
f(n)=g(n)+h(n)
\]

其中，`g(n)` 表示从起点到当前节点 `n` 的实际代价，`h(n)` 表示从当前节点到目标节点的估计代价。

## 3.2 改进总体框架
本文提出的改进 A* 包括四个模块：
1. Octile 启发函数替代传统启发估计；
2. 基于障碍率的启发权重自适应机制；
3. 融合 JPS-like 的搜索加速策略；
4. 两阶段路径平滑策略（删点 + 拐角平滑）。

整体流程为：地图输入 -> 改进搜索 -> 路径回溯 -> 一次平滑 -> 二次平滑 -> 输出最终路径。

## 3.3 Octile 启发函数
在 8 邻域栅格中，采用 Octile 距离可更好匹配斜向移动代价。设
`dx=|xg-xn|`，`dy=|yg-yn|`，则：

\[
h_{oct}(n)=\max(dx,dy)+(\sqrt{2}-1)\min(dx,dy)
\]

相较欧氏距离与曼哈顿距离，Octile 距离在 8 邻域下更接近真实网格最短路代价，可减少无效扩展。

## 3.4 启发权重自适应机制
为平衡搜索速度与路径质量，构建加权评价函数：

\[
f(n)=g(n)+\alpha \cdot h_{oct}(n)
\]

其中 `alpha` 随障碍率变化。当障碍率较低时，提高启发项权重以提升搜索效率；当障碍率较高时，降低启发项权重以避免过度贪心。本文采用障碍率驱动的单调函数 `alpha=f(Os)`（例如对数映射）实现自适应调节。

## 3.5 JPS-like 融合搜索策略
传统 A* 在复杂地图中会扩展大量冗余邻居节点。为此，本文在扩展阶段引入 JPS 思想的轻量实现（JPS-like）：沿 8 个方向执行有限步长跳跃扩展（当前实现默认最大步长为 3），以减少中间冗余节点访问。该策略通过降低 OpenList/CloseList 规模缩短搜索时间，尤其在中大尺度地图中效果更明显。

## 3.6 两阶段路径平滑

### 3.6.1 一次平滑：冗余点删除
对回溯得到的离散路径执行两类简化：
1. 删除共线中间点；
2. 对满足可视线直连条件的路径段进行跨点连接，减少不必要拐点。

### 3.6.2 二次平滑：拐角曲线化
在不与障碍碰撞的约束下，对尖锐拐角进行离散平滑（邻域加权重定位），降低局部方向突变，提高路径可跟踪性与运动平顺性。

## 3.7 算法伪代码（简版）
```text
Input: grid_map, start, goal
Output: smoothed_path

1: init OpenList, CloseList
2: while OpenList not empty:
3:     n <- pop node with min f
4:     if n == goal: break
5:     expand by JPS-like neighbors (bounded jump steps)
6:     for each neighbor m:
7:         h <- Octile(m, goal)
8:         alpha <- adaptive_weight(obstacle_ratio)
9:         f(m) <- g(m) + alpha * h
10:        update OpenList
11: path <- backtrack(goal)
12: path1 <- primary_smoothing(path)
13: path2 <- secondary_smoothing(path1)
14: return path2
```

## 3.8 复杂度与预期收益
理论上，JPS-like 融合可减少节点扩展数量；自适应权重可在不同障碍率场景下平衡最优性与效率；两阶段平滑可降低路径离散抖动。综合预期表现为：运行时间下降、转弯次数下降，路径长度在多数场景呈下降趋势。
