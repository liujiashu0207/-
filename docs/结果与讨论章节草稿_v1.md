# 第5章 结果与讨论（草稿 v1）

## 5.1 实验结果总览
在修正地图采样连通性并固定每组 30 个有效样本后，各场景均获得可复核结果。总体上，改进 A* 在路径质量指标（尤其转弯次数）上表现稳定优势，在运行时间上相对传统 A* 与 Dijkstra 在多数组景有明显下降。

## 5.2 低障碍率场景（30%）分析
在 `20x20、40x40、80x80、100x100` 四组地图中，改进 A* 相比传统 A* 在均值层面取得更短路径和更少转弯，且运行时间在中大地图中下降明显。以 `100x100, 30%` 为例：
- 传统 A* 运行时间均值为 `1.5559 ms`，改进 A* 为 `0.6532 ms`，约下降 `58.0%`；
- 路径长度由 `63.2640` 降至 `60.2467`，约下降 `4.77%`；
- 转弯次数由 `20.47` 降至 `13.37`，约下降 `34.7%`。

结合置换检验结果，运行时间和转弯次数在部分场景达到统计显著，路径长度改进当前主要体现为均值趋势优势。

## 5.3 高障碍率场景（70%）分析
在高障碍场景下，改进 A* 与其他算法均保持 `100%` 成功率（在可达采样条件下）。改进 A* 在路径质量上保持均值优势。以 `100x100, 70%` 为例：
- 路径长度由 `8.4340` 降至 `7.2827`，约下降 `13.65%`；
- 转弯次数由 `3.87` 降至 `2.40`，约下降 `37.9%`。

运行时间方面，改进 A* 在该组并非最优（加权 A* 更快），但其路径质量更高，体现了“速度-质量”平衡策略的价值。

## 5.4 与各基线方法的对比讨论
1. 与 Dijkstra 相比：改进 A* 在运行时间与扩展节点数上优势显著，尤其在大图中更明显。  
2. 与传统 A* 相比：改进 A* 在路径长度、转弯次数、运行时间三项主指标上实现综合改进。  
3. 与加权 A* 相比：加权 A* 在部分场景时间更短，但路径更长、转弯更多；改进 A* 更适合作为机器人执行路径。  
4. 与 JPS-like 相比：改进 A* 借助平滑与自适应权重，在路径质量上更优，综合指标更均衡。

## 5.5 结果机制解释
改进 A* 的性能提升来源于四个模块的协同：
- Octile 启发在 8 邻域中提供更贴近实际代价的估计；
- 自适应权重在不同障碍率下动态平衡搜索贪心程度；
- JPS 融合减少冗余扩展节点；
- 两阶段平滑直接降低折线路径中的冗余转折。

## 5.6 局限性与后续优化
1. 本文当前版本主要验证静态栅格场景，动态障碍情形尚未系统展开。  
2. JPS-like 为工程简化实现，后续可替换为更完整的双向 JPS 或标准 JPS 剪枝流程。  
3. 未来可补充 `Theta*` 或 `D*Lite` 作为更强基线，进一步提高审稿说服力。  
4. 可在 ROS/Gazebo 场景中进行实机或半实物验证，增强工程落地性。

## 5.7 小结
实验结果表明，改进 A* 在多尺度、多障碍率栅格地图中实现了路径质量与规划效率的平衡提升，具备进一步扩展到真实移动机器人导航任务的潜力。
