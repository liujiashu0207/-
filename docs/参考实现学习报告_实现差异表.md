## 一、算法规则卡片

### A* (PathFinding.js 参考标准)
| 项目 | 规则 |
|------|------|
| 输入 | 栅格地图, 起点, 终点, 对角模式, 启发函数, weight |
| 输出 | 最短路径（weight=1 且 h 可采纳时保证最优） |
| 启发函数 | 开启对角移动后**自动切换为 Octile**（代码注释："manhattan is not admissible"） |
| 权重 | `opt.weight`，全局常量，默认 1。节点的 h 值**只算一次然后缓存** |
| 对角模式 | 4 种：Never / IfAtMostOneObstacle(OR) / OnlyWhenNoObstacles(AND) / Always |
| g 值 | 正交=1, 对角=√2 |

### Dijkstra
| 项目 | 规则 |
|------|------|
| 等价于 A* with h=0 | 邻居生成规则与 A* 共享 |

### Weighted A*
| 项目 | 规则 |
|------|------|
| f(n) = g(n) + weight × h(n) | weight > 1 时搜索加速但路径 ≤ weight × 最优 |
| PathFinding.js | weight 是全局常量，每节点 h 缓存 |
| 我们的创新 | **节点级动态 α(x,y)**——参考实现中无先例 |

### JPS (PathFinding.js + ViktorRubenko)
| 项目 | 规则 |
|------|------|
| 核心 | 递归 `_jump` 沿方向无限跳跃，直到跳点或撞墙 |
| 对角跳跃前提 | PathFinding.js `OnlyWhenNoObstacles` 变体：两个正交邻居都可通行才允许对角 |
| 输出 | 与 A* 相同的最优路径（加速搜索不改变结果） |
| 关键 | **不做逐步 line_of_sight**——跳跃过程中只检查目标格可通行性 |

### DWA (PythonRobotics)
| 项目 | 规则 |
|------|------|
| 输入 | 机器人状态(x,y,θ,v,ω), 目标, 障碍物列表 |
| 输出 | 速度指令(v,ω)，不是路径 |
| 性质 | **局部规划器**，连续空间采样，与 A* 互补不可直接对比 |

### D* Lite (mdeyo/Sollimann)
| 项目 | 规则 |
|------|------|
| 性质 | 增量式重规划，适用于动态地图 |
| 参考实现 | 4 连通（无对角线），边权均为 1 |
| 与本文关系 | 论文仅在文献综述和"未来工作"中提及，不做实验对比 |

## 三、与我们现有代码的差异表

### 3.1 一致的（安全）

| 逻辑项 | 参考标准 | 我们的实现 | 结论 |
|--------|---------|-----------|------|
| anti-tunneling 对角检测 | PathFinding.js `OnlyWhenNoObstacles`：AND 两个正交都可通行 | `grid[x+dx,y]==1 or grid[x,y+dy]==1` 则禁止 | ✅ 一致（最严格模式） |
| 步长代价 | 正交=1, 对角=√2 | 正交=1.0, 对角=math.sqrt(2.0) | ✅ 一致 |
| Octile 启发函数 | `F*min(dx,dy) + max(dx,dy)` 其中 F=√2−1 | `max(dx,dy) + (√2−1)*min(dx,dy)` | ✅ 数学等价 |
| 开放列表 | 二叉堆 | heapq 二叉堆 | ✅ 一致 |
| 关闭集 | 节点标记 | set() | ✅ 功能一致 |
| g 值更新 | `if ng < neighbor.g` 则更新 | `if tentative_g < g_score.get(nb, inf)` 则更新 | ✅ 一致 |
| MovingAI scen 坐标 | (x=col, y=row), (0,0)=左上角 | `start=(sr,sc)` → (row,col) | ✅ 一致（已验证） |
| 最优路径长度 | "cannot cut corners" + √2 对角 | 10/10 任务长度精确匹配 | ✅ 完全一致 |

### 3.2 不一致但合理的（需论文说明）

| 逻辑项 | 参考标准 | 我们的实现 | 差异原因 | 风险等级 |
|--------|---------|-----------|---------|---------|
| **基线A*的启发函数** | PathFinding.js: 8邻域自动用Octile | 我们: 基线用**欧氏距离** | 为了让Octile成为创新点 | 🟡 审稿人可能问"为什么基线不用Octile" |
| **权重机制** | 所有参考: weight是全局常量 | 我们: **节点级动态α(x,y)** | **核心创新点** | 🟢 这正是新贡献 |
| **路径平滑** | PathFinding.js/PythonRobotics均无 | 我们: 两阶段平滑 | **创新模块** | 🟢 无冲突 |
| **障碍物膨胀** | PythonRobotics: 按robot_radius膨胀 | 我们: 无膨胀（点机器人） | 简化假设 | 🟡 论文需说明"假设点机器人" |

### 3.3 不一致且需要注意的（潜在问题）

| 逻辑项 | 参考标准 | 我们的实现 | 穿模风险 | 建议处理 |
|--------|---------|-----------|---------|---------|
| **ViktorRubenko JPS的对角规则** | AND-to-block（仅当两个正交都是障碍才禁止对角） | 我们用OR-to-block（任一正交是障碍就禁止） | 无风险——我们更严格 | 无需改，论文说明采用最严格模式 |
| **c2huc2hu JPS** | 明确标注"allows corner cutting" | 我们禁止corner cutting | 无风险——我们更严格 | 与MovingAI口径一致 |
| **PythonRobotics A*的对角移动** | 无anti-tunneling检测（靠半径膨胀） | 我们有anti-tunneling | 无风险——我们更严格 | 无需改 |
| **节点级α的理论性质** | Weighted A*保证α-最优（weight×optimal） | 节点级变化的α不保证全局α-最优 | **不是穿模风险，是理论性质降级** | 论文必须说明：不保证严格α-最优，但实验证明路径质量可控（路径长度改善4-5%而非恶化） |

| 逻辑项 | 状态 | 说明 |
|--------|------|------|
| `_jump_like_neighbors` | 已关闭（`use_jump_like=False`） | 不是真JPS，不参与任何实验 |
| `jps_like_search` | 函数存在但不在实验中调用 | 可保留代码，实验不使用 |
